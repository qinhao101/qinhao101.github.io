下载安装nodejs

推荐到nodejs官网下载
下载后得到node-v16.13.1-linux-x64.tar.xz，解压，记住所在位置
使用ln命令在/usr/local/bin/下创建可执行文件的链接

不建议使用apt-get install 的方式安装node和npm，装好后会出现npm和node版本不匹配的各种问题

hexo配置

hexo有两份配置文件，名称都是_config.yml，其中一份在站点的根目录下，比如我们的~/myblog下面的_config.yml，主要用于hexo自身的配置；另一份在themes目录下面的每个主题目录中，用于配置主题相关的选项；按照next的使用文档，把前者称为站点配置文件，后者称为主题配置文件

以next主题为例，更多hexo主题可以在hexo官网下载
next主题还有很详细的设置教程，具体参见教程

下载主题

启用主题

想要启用下载好的主题，只需要在站点配置文件中找到 theme字段，将起改为主题对应的文件夹名字即可

next文档中关于语言的说明

next文档中简体中文为zh-Hans，但是主题的language包中并没有这个语言，所以站点配置文件language字段设置成zh-Hans后，网页不能显示中文；简单的方法就是把language字段设置成语言包中有的语言即可，比如zh-CN，或者可以自己创建一个zh-Hans的md文件，查看其它语言的md文件可以看到里面都是网站上所有单词对应的相应翻译，内容不多自己创建也很简单。

配置ssh key

密码设置为空，一路回车键下来，默认在家目录的.ssh文件夹下生成id_rsa和id_rsa.pub文件
进入github主页的个人设置->SSH and GPG keys -> New SSH key：
把id_rsa.pub文件的内容粘贴到key里面，title随便填写，然后保存

hexo 进阶

hexo布局

创建文章时默认使用的时post布局，所以会在source/_posts 里生成相应的md文件
hexo有三种不同的布局，对应不同的存储位置
| 布局 | 路径 |
| —– | ————- |
| post | source/_posts |
| page | source |
| draft | source/draft |

new这个命令其实是对应的
